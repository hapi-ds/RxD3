# Type Generation Scripts

This directory contains scripts for generating TypeScript types from backend Python models.

## generate-types.ts

Automatically generates TypeScript interfaces and enums from backend Pydantic models.

### Usage

```bash
npm run generate-types
```

### What it does

1. Parses Python files from `backend/src/models/`:
   - `enums.py` - Enum definitions
   - `mind.py` - Base Mind model
   - `mind_types.py` - Specialized Mind types

2. Generates TypeScript types in `src/types/generated.ts`:
   - Enums matching backend enums
   - Interfaces for each Mind type
   - Union types (Mind, NodeType)

3. Maps Python types to TypeScript types:
   - `str` → `string`
   - `int`, `float` → `number`
   - `bool` → `boolean`
   - `date`, `datetime` → `string` (ISO 8601)
   - `UUID` → `string`
   - `Optional[T]` → `T | null`
   - `list[T]` → `T[]`
   - Enum types preserved

### When to run

Run this script whenever backend models change to keep frontend types in sync.

### Requirements

- Backend Python models must be available at `../../../backend/src/models/`
- Node.js and npm installed
- tsx package (installed as dev dependency)

### Output

Generated file: `src/types/generated.ts`

**Do not edit this file manually** - it will be overwritten on the next generation.
